# Лог архитектурных решений

Этот файл содержит важные архитектурные и технические решения принятые в проекте.

**ИИ автоматически добавляет записи** когда:
- Выбран архитектурный паттерн
- Выбрана библиотека/фреймворк из нескольких вариантов
- Принято решение о структуре проекта
- Изменён подход к тестированию/деплою
- Принято важное техническое решение

---

## Формат записи

```markdown
## YYYY-MM-DD: Краткое название решения

**Контекст**: Почему возникла необходимость принять решение

**Решение**: Что решили делать

**Альтернативы**: Какие варианты рассматривали и почему отклонили

**Последствия**: К чему это приведёт (плюсы/минусы)
```

---

## 2025-12-27: Rules sync as source of truth

**Контекст**: Нужна единая база правил для разных ассистентов без рассинхрона.

**Решение**: Ввести `rules/` как источник истины и синхронизировать правила в
`AGENTS.md`, `INSTRUCTIONS.md`, `CLAUDE.md`, `.cursorrules`, `.antigravity/rules.md`.

**Альтернативы**: Оставить разрозненные правила в каждом файле вручную;
обновлять только `INSTRUCTIONS.md` и `AGENTS.md`.

**Последствия**: Меньше дрейфа между ассистентами, но требуется поддерживать
синхронизацию правил.
