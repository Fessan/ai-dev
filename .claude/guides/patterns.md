## Тестирование

### Подход
**TDD обязателен** - всегда пишем тест перед кодом.

### Процесс (RED-GREEN-REFACTOR)
1. Написать **падающий тест**
2. Запустить тест - убедиться что **падает**
3. Написать **минимальный код** для прохождения
4. Запустить тест - убедиться что **проходит**
5. Рефакторинг (если нужен)
6. Коммит

## Git workflow

### Стиль коммитов
[Conventional Commits / другой]

**Формат**:
```
type(scope): subject

body
```

**Типы коммитов**:
- `feat`: новая фича
- `fix`: баг-фикс
- `refactor`: рефакторинг
- `test`: добавление/изменение тестов
- `docs`: документация

### Ветки
- `main` / `master` - production
- `develop` - разработка (если используется)
- `feature/*` - фичи
- `fix/*` - баг-фиксы
