# /plan

Создать детальный план реализации для фичи.

## Описание

Создаёт детальный план с разбивкой на задачи на основе ПРД.

## Использование

```
/plan work/2025-01-15--auth/prd.md
```

## Что делает

1. Читает ПРД
2. Читает guides (architecture, patterns)
3. Анализирует существующий код
4. Создаёт детальный план с задачами
5. Разбивает на небольшие задачи (по 15-30 минут каждая)
6. Определяет зависимости между задачами
7. Создаёт файлы задач в `work/<feature>/tasks/`

## Output

Создаёт:
- `work/<feature>/plan.md` - общий план
- `work/<feature>/tasks/task-001.md` - задача 1
- `work/<feature>/tasks/task-002.md` - задача 2
- и т.д.

## Принципы

- **Декомпозиция**: маленькие задачи лучше больших
- **Детальность**: каждая задача должна быть понятна разработчику
- **Зависимости**: явно указывай зависимости
- **Проверяемость**: каждая задача должна иметь acceptance criteria
