# /dev

Запустить разработку задачи с TDD.

## Описание

Запускает агента `code-developer` для реализации задачи.

## Использование

```
/dev work/2025-01-15--auth/tasks/task-001.md
```

или просто:

```
/dev
```

(агент спросит какую задачу брать)

## Что делает

1. Запускает агента `code-developer`
2. Агент читает задачу, ПРД, guides
3. Пишет тесты (TDD)
4. Пишет код
5. Запускает тесты
6. Проверяет acceptance criteria
7. Возвращает отчёт
8. Спрашивает можно ли взять следующую задачу

## Workflow

```
User: /dev work/2025-01-15--auth/tasks/task-001.md

AI: Запускаю code-developer для задачи task-001...

[Agent работает]

Agent:
## Implementation Report
**Статус**: SUCCESS
**Задача**: Создать модель User
**Тесты**: PASSED ✅
...

AI: Задача выполнена! Все тесты зелёные.
Взять следующую задачу (task-002)?
```

## Принципы

- **TDD обязателен**: всегда тест → код
- **По одной задаче**: не берём несколько задач сразу
- **Спрашиваем**: после каждой задачи спрашиваем о следующей
- **Автообновление**: guides обновляются автоматически
