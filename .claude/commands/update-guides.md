# /update-guides

Обновить guides вручную.

## Описание

Guides обычно обновляются автоматически, но эта команда позволяет обновить их вручную.

## Использование

Обновить все guides:
```
/update-guides
```

Обновить конкретный guide:
```
/update-guides architecture
```

## Какие guides

- `project` - project.md
- `architecture` - architecture.md
- `features` - features.md
- `patterns` - patterns.md
- `business-rules` - business-rules.md

## Что делает

1. Анализирует текущее состояние проекта
2. Читает существующие файлы, зависимости, структуру
3. Обновляет соответствующий guide
4. Показывает что изменилось

## Когда использовать

- После крупных изменений в архитектуре
- Когда guides устарели
- После ручных изменений в коде без автообновления
- Для синхронизации guides с реальностью

## Output

```markdown
## Guides Updated

### architecture.md
Изменения:
- Добавлена зависимость: redis 5.0.0
- Обновлена структура папок
- Добавлена команда запуска celery worker

### features.md
Изменения:
- Добавлена фича: User Authentication
- Обновлён статус: Email Notifications → Реализовано

---

✅ Guides обновлены и синхронизированы с проектом
```

## Принципы

- **Анализ кода**: читай реальные файлы, не предполагай
- **Точность**: guides должны отражать реальность
- **Полнота**: не пропускай важные детали
