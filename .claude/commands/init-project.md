# /init-project

Быстрая инициализация проекта через короткое интервью.

## Процесс

### 1. Приветствие

```
Привет! Настроим проект быстро.
Отвечай коротко, я заполню context.md.
```

### 2. Вопросы (5-7 штук)

**1. Название и описание?**
```
Что за проект? (название + 1 предложение)
Пример: "TaskBot — телеграм-бот для управления задачами"
```

**2. Для кого и какую проблему решает?**
```
Пример: "Для фрилансеров, сложно трекать задачи"
```

**3. Стек?**
```
Язык + фреймворк + БД (если нужна)
Пример: "Python, FastAPI, PostgreSQL"

Не знаешь — рекомендую:
- Бот: Python + aiogram
- API: Python + FastAPI
- Fullstack: TypeScript + Next.js
```

**4. Есть особые требования?**
```
Интеграции, ограничения, бизнес-правила?
Пример: "Интеграция с Stripe" или "нет"
```

### 3. После ответов

1. **Заполни `context.md`** на основе ответов

2. **Создай базовую структуру** (если её нет):

Python:
```
src/
├── __init__.py
├── main.py
└── config.py
tests/
└── test_main.py
requirements.txt
.env.example
```

TypeScript:
```
src/
├── index.ts
└── config.ts
tests/
└── index.test.ts
package.json
.env.example
```

3. **Настрой git** (если нет `.git`):
```bash
git init
git add .
git commit -m "feat: initial setup"
```

### 4. Финальный отчёт

```markdown
Готово!

## Создано
- context.md — заполнен
- src/ — базовая структура
- tests/ — папка для тестов
- .env.example — шаблон переменных

## Следующие шаги
1. Установи зависимости: [команда]
2. Скопируй .env: cp .env.example .env
3. Начинай: "Хочу добавить [фичу]"
```

## Принципы

- Минимум вопросов, максимум действий
- Рекомендуй если пользователь не знает
- Создавай рабочий проект, не шаблон
